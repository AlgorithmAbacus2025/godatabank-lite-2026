# GoDataBank Agent Laws

These laws apply to every human or automated agent that collects, verifies, organises, summarises, or publishes material for GoDataBank.

Truth is retrieved, not generated.

## Law 1: Official Sources Only

Use official institutional sources only. A source must be issued by a recognised public, governmental, regulatory, statistical, judicial, standards, central-bank, or intergovernmental institution.

## Law 2: Machine-Readable Access Required

Do not admit a source unless it provides a public API or approved machine-readable feed.

## Law 3: English Content Required

Do not admit a source unless the relevant content or metadata is available in English.

## Law 4: Citation Metadata Is Mandatory

Every admitted source must include citation metadata before it can support any report output.

Required citation metadata includes:

- Issuing institution
- Source title or dataset name
- Source URL
- API, feed, or structured-file URL where applicable
- Publication date, release date, or version date where available
- Retrieval timestamp
- Version, edition, table, endpoint, field, or query reference where applicable
- Language
- Verification status

## Law 5: No Excluded Source Categories

Never use news organisations, commercial data vendors, think tanks, blogs, social media, user-generated content, advocacy publishers, personal sites, forums, or unofficial aggregators as evidence sources.

## Law 6: No Unsourced Output

Do not publish claims, figures, rankings, summaries, tables, charts, or conclusions without a cited admitted source.

## Law 7: No Speculation

Do not speculate. If source evidence is absent, incomplete, or not admitted, say that no sourced output is available.

## Law 8: No Opinion

Do not publish opinion, preference, editorial judgement, or interpretation unsupported by admitted source evidence.

## Law 9: No Source Match, No Output

If no admitted source matches the requested claim, question, geography, time period, metric, or definition, produce no claim.

## Law 10: Verification Before Summary

Summaries must be based on verified admitted source material. If the source cannot be verified, the correct output is a limitation or rejection, not a summary.

## Law 11: Preserve Units and Methods

Numerical outputs must preserve units, time periods, geography, definitions, and calculation methods. Derived figures must show their inputs and formula.

## Law 12: State Uncertainty

If data is incomplete, unavailable, ambiguous, outdated, or inconsistent, state that limitation plainly. Do not hide uncertainty behind confident wording.

## Law 13: No Publication Without Validation

Do not publish until source admission, citation metadata, exclusion checks, and claim-level traceability have been validated.

## Law 14: Stay Inside Scope

Do not create API connectors, AI summarisation code, databases, subscription features, or new product areas unless a later milestone explicitly authorises that work.
